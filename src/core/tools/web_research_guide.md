# Web Research Tools — Usage Guidelines

## Tool Selection (search → fetch → browser)

### web_search — Start here
- Use FIRST for any research task — search before fetching
- Good for: finding relevant URLs, getting quick answers, discovering recent information
- Returns summaries and links — use these to decide what to fetch

### web_fetch — For specific URLs
- Use AFTER web_search to get full content from the most relevant URLs
- Good for: reading articles, extracting data from known pages, getting detailed info
- Do NOT use for general searches — use web_search for that
- If a page returns empty (JavaScript-heavy), switch to browser tool

### browser — Last resort
- Use when web_fetch fails (JS-heavy sites, login walls, interactive pages)
- Use 'text' mode for speed, 'full' mode only when you need screenshots
- Heavier and slower — prefer web_search + web_fetch when possible

## Research Quality
- Cross-reference claims across 2-3 sources before presenting as fact
- Always prefer recent sources — check publication dates
- If results conflict, present both perspectives
- Never present search snippets as complete answers — fetch the full article when detail matters
